# Improvement Output Template

Use this template as a reference for the expected output structure of the plan quality improver.

## Structure

The output follows this exact format:

```markdown
# Improved Plan Document

## Change Summary
- High-level overview of what was improved
- Number of clarity improvements
- Number of acceptance criteria enhancements
- Number of edge case considerations added

## Revised Plan
(Provide the full revised plan with improvements applied)

## Improvement Rationale
- Bullet list explaining why each category of change was made
- Reference corresponding issues from the analysis report if provided
```

## Improved Plan Guidelines

- Preserve all original plan structure and major milestones
- Only modify sections that were flagged in the analysis report
- Apply changes systematically based on issue categories
- Maintain original document structure unless structural improvements were recommended

## Change Summary Guidelines

- Provide a high-level overview of the improvements made
- Count the number of changes in each category:
  - Clarity improvements
  - Acceptance criteria enhancements
  - Edge case additions
  - Consistency fixes
- Keep the summary concise but informative

## Improvement Rationale Guidelines

- Explain the reasoning behind each category of change
- Reference specific issues from the analysis report when available
- Use consistent improvement type classifications:
  - Clarity: Language clarification, ambiguity resolution
  - Acceptance: Criteria addition or improvement
  - Edge Case: Missing scenario addition
  - Consistency: Terminology or formatting fixes
  - Structural: Organization or flow improvements
- Provide concise but meaningful explanations
- Maintain traceability for audit purposes
